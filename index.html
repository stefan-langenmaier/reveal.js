Ici le monde a réussi. Maintenant ma mère et capable d'envoyer un smilie quand elle veux de n'importe ou. Ici je parle des silos mechant/férme, mais aussi de silo ouvert comme wikipedia. Qu'est-ce que le semantic web? Un web qui est enrichi avec des metadonné pour permettre les machine à comprendre le web. Ça veux dire les metadonnés sont des information supplementaires spécifier par les humaines pour aider les ordinateurs/logiciel avec la comprehension. In the beginning of the buzzword time for Web 2.0 the following terms were of equal importance * mash-ups * folksonmies * metadata markup/semantic web * social * ... We mostly got the social part, but with the cost of losing the decentralization. Mash-ups were hyped for some time, but they not really lived up to the potential, the semantic web was buried in proprietary silos. http://de.wikipedia.org/wiki/Web_2.0

Tous les approche présenter utilise HTML comme base

Ils veulent eviter d'utiliser un deuxieme format. Le but est de rutiliser les outils existants. Les outils pour créer, publier et stocker les information. Au même temps les informations sont à côté les informations humain. Les differences entre les approches c'est quoi est possible à embedder et comment c'est fait. L'ordre de point est un peu l'ordre historique et c'est aussi l'ordre dans quelle on va les regarder. Their common apporach is to use HTML as basis for the embedding extra data. They want to avoid using a special format. Their goal is on reusing existing tools. There were a lot of people who saw these problems and tried to solve the problems. Reusing already existing methods for publishing and augmenting already exisiting data. Differences is what they can embed and how.

Microformats

  • l'approche le plus vieux
  • rutiliser attribus HTML
  • adapter les specification existant

L'autorité central est sur : http://microformats.org/

C'est l'approche le plus vieux. Il rutilisent les attribut standard dans le balise HTML et donne les un semantique speciale. Pour les formats ils rutilisent les specification existants. L'autorité central est sur : http://microformats.org/

They are the oldest approach to embed metadata in the HTML attributes They are taking existing formats and specifications and simplify them to be used in HTML attributes. The central authority is the following website: http://microformats.org/

Suivant les nom des specifications pour un format de metadonné va être vocabulaires. Il y a plusieurs et dans la lingo de microformat c'est "profile", mais pour être consistent on va utiliser vocabulaire

Pour la prochaine section je vais commencer par RDFa lite, parce que c'est un vrai sous-ensemble de RDFa Donc, c'est quoi. C'est aussi un standard de W3. C'est un peu plus claire comme le Microdata, la recherche de Google soutien aussi RDFa lite, tous le vocabulaires de schema.org sont compatible, mais il y a aussi beaucoup des autres vocabulaires déjà fait pour RDF. Et c'est aussi dessiner pour HTML5, c'est la raison pourquoi ça existes seulement depuis 2012. RDFa lite cotien cinq attributs * vocab * typeof * property * resource * prefix On va les regarder dans les examples suivants. RDFa Lite consists of five simple attributes; vocab, typeof, property, resource, and prefix.

RDFa lite - Examples

<p vocab="http://schema.org/" typeof="Person">
   My name is
   <span property="name">Manu Sporny</span>
   and you can give me a ring via
   <span property="telephone">1-800-555-0199</span>.
   <img property="image" src="http://manu.sporny.org/images/manu.png" />
   My favorite animal is the Liger.
</p>
Ici on peut voir l'information concernant une personne. On veut ajouter les metadonnées. Les premiers deux choses ses sont les attributs pour le context. vocab ça define le default pour la base de type suivant et le type est une personne. on pourrais mettre aussi l'url absolu dans le type. Pour les valeurs, ça fonctionne avec le même principe comme pour le microdata. il y a des properties differents.

RDFa lite - Examples

<p vocab="http://schema.org/"
  prefix="ov: http://open.vocab.org/terms/" resource="#manu"
  typeof="Person">
   My name is
   <span property="name">Manu Sporny</span>
   and you can give me a ring via
   <span property="telephone">1-800-555-0199</span>.
   <img property="image" src="http://manu.sporny.org/images/manu.png" />
   My favorite animal is the
   <span property="ov:preferredAnimal">Liger</span>.
</p>

Dans ce cas là c'est la open vocab avec le properties preferredAnimal. Et pour completer j'ai ajouter la cinquième attribut resource. Avec ça on peut nommer une section de metadonné. RDFa

  • standard officiel de W3
  • contient RDFa lite
  • au début seulement disponible pour XHTML
On va continuer directement avec RDFa C'est aussi un standard officiel de W3 et c'est complètement compatible avec RDFa lite. Ça veut dire que chaque markup de RDFa lite est automaitqument RDFa. Si on utilise des attributs pour RDFa le markup devient automatiquement RDFa. Ça veux dire c'est aussi facile a chager entre les deux. La difference le plus grande est dans l'histoire. Au début de RDFa c'était seulement disponible pour XHTML ou en general les language XML comme par exemple SVG. Ça fonctionne encore aujourd'hui mais avec la dernière specification c'est aussi defini pour HTML5. RDFa 1.1 Lite is completely upwards compatible with the full set of RDFa 1.1 attributes. This means that if an author finds that RDFa Lite isn't powerful enough, transitioning to the full version of RDFa is just a matter of adding the more powerful RDFa attributes into the existing RDFa Lite markup. Historically interwoven with RDFa, RDFa is older (like microformats but was only designed for xml/xhtml) that changed only very recently june 2012 Already existed for a longer time but back then only for XML and with the XHTML. Now with RDFa 1.1 June 2012 it is also specified for HTML5. It contains all the things already discussed for RDFa lite.
